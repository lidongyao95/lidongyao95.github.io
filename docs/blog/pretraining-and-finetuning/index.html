模型训练范式：预训练与微调到底在做什么

前言

今天的大模型很少是「从零开始为某个任务训练」出来的。更常见的路线是：

海量通用数据 → 预训练基础模型 → 针对任务微调 → 部署到具体场景

这条路线听起来简单，但里面有一个很关键的思想转变：模型不是先学某个具体任务，而是先学世界中大量可迁移的模式，再用少量数据把能力对齐到目标任务。

这篇文章从最底层的问题开始：预训练到底学了什么？微调到底改了什么？为什么这套范式能让模型在很多任务上表现得更好？

1. 为什么需要预训练

假设你要训练一个文本分类模型，目标是判断一句评论是正面还是负面。

如果只有 1000 条标注数据，从零训练一个神经网络会遇到两个问题：

  • 数据太少：模型很容易记住训练样本，而不是学到真正的语言规律。
  • 任务太窄：情感分类数据只告诉模型「这句话是正面还是负面」，却没有教它语法、常识、指代关系和上下文理解。

人类学习也不是从「情感分类题库」开始的。我们先通过大量阅读和交流获得语言能力，然后才学习某个具体任务的判断标准。

预训练就是这个思路的机器版本：先让模型在大规模通用数据上学习基础表示，再把这种表示迁移到下游任务。

2. 预训练：先学通用能力

预训练的核心是用海量数据训练一个基础模型。这里的数据通常不需要人工逐条标注，因为训练信号可以从数据本身构造出来。

2.1 语言模型的预训练目标

以自回归语言模型为例，训练目标非常直接：

给定前面的词，预测下一个词。

例如：

输入：机器学习的核心目标是让模型具备
目标：泛化

模型每次预测下一个 token，如果预测错了，就通过反向传播更新参数。这个过程重复数万亿次后，模型为了降低预测错误，不得不学会很多隐藏规律：

  • 词语之间的搭配关系
  • 句法结构和长距离依赖
  • 常见事实和世界知识
  • 推理步骤的文本模式
  • 不同文体和领域的表达方式

看起来只是「猜下一个词」，本质上却是在压缩数据分布。一个能持续预测文本的模型，必须在参数中存下大量关于语言和世界的统计结构。

2.2 表示学习：把输入变成可用的内部结构

预训练不只是让模型背知识，更重要的是学到一种表示能力。

一句话进入模型后，会被转成一层层向量表示。浅层可能更关注词形和局部搭配，深层可能更关注语义、关系和任务线索。训练越充分，这些向量越能把「表面不同但意义相近」的输入放到相近的位置。

比如：

这家店味道不错
下次还会再来
服务很糟糕

一个预训练好的模型，即使没专门学过情感分类，也往往已经能把前两句和第三句区分开，因为它在大量文本中见过类似表达的上下文。

3. 微调：把通用模型推向具体任务

预训练模型像一个知识面很广但还没有岗位说明书的人。微调就是给它具体任务、评价标准和输出格式。

3.1 监督微调

最经典的微调方式是监督微调。我们准备一批输入和期望输出：

输入期望输出
这个手机续航很强正面
包装破损，客服也不处理负面
请总结下面这段文字一段摘要

训练时，模型先根据输入给出预测，再和期望输出计算损失，然后更新参数。

这一步会改变模型的行为倾向：

  • 更熟悉任务的输入形式
  • 更稳定地产生目标格式
  • 更重视领域里的关键特征
  • 减少和任务无关的输出

如果预训练是在建立「语言和世界的底座」，微调就是在这个底座上安装「任务接口」。

3.2 全量微调与参数高效微调

微调不一定要更新模型的全部参数。

全量微调会更新所有权重，优点是适应能力强，缺点是成本高、显存需求大，也更容易破坏原有能力。

参数高效微调只训练少量新增参数或低秩矩阵，例如 LoRA。它的思想是：基础模型里已经有大量可用能力，下游任务往往只需要一个小的方向调整。

可以把基础模型想象成一张很复杂的地图。全量微调是重画整张地图，LoRA 更像是在地图上加几条新的路线标注。

4. 指令微调：让模型学会听懂任务

大语言模型还有一种非常重要的微调：指令微调。

普通预训练模型擅长续写文本，但不一定知道用户说「请解释这段代码」时应该真的解释代码。它可能继续补全一段类似网页、论坛或论文的文本。

指令微调会使用大量这样的样本：

用户：用通俗语言解释什么是梯度下降。
助手：梯度下降可以理解为在山坡上找最低点……

模型由此学会：

  • 识别用户意图
  • 按要求组织答案
  • 遵守格式约束
  • 在对话中保持上下文

这也是为什么基础模型、指令模型和聊天模型的体验差别很大。它们可能拥有相似的底层知识，但对「如何响应人类请求」的训练程度不同。

5. 预训练和微调的区别

可以从四个维度理解它们：

维度预训练微调
数据规模极大较小
数据类型通用文本、代码、多模态数据任务数据、领域数据、指令数据
训练目标学习通用分布和表示适配具体任务或行为
参数变化从随机参数学到底座能力在已有能力上定向调整

最关键的区别是：预训练提供能力上限，微调决定能力如何被调用。

一个底座很弱的模型，微调再精细也很难突然变强；一个底座很强的模型，如果没有合适微调，也可能输出不稳定、不听指令或不符合业务格式。

6. 微调不是万能的

很多人把微调理解成「给模型灌知识」，这并不准确。

微调更擅长改变行为模式，而不是大规模注入事实知识。如果目标是让模型掌握大量实时或私有资料，通常还需要结合检索增强生成（RAG）、工具调用或外部数据库。

微调适合解决的问题包括：

  • 固定输出格式
  • 特定语气和角色
  • 领域任务的判断标准
  • 少量稳定的专业概念
  • 对内部流程的响应规范

不太适合只靠微调解决的问题包括：

  • 频繁变化的事实
  • 大规模私有知识库问答
  • 需要严格可追溯来源的答案
  • 高精度数值计算和实时查询

工程上常见的组合是：预训练模型提供通用能力，RAG 提供外部知识，微调提供稳定的任务行为。

7. 一个直观例子

假设我们要做一个医疗问答助手。

预训练阶段让模型学会语言、医学文本的常见表达、基础推理结构和问答模式。

指令微调阶段让模型学会回答问题、拒绝不安全请求、说明不确定性。

领域微调阶段让模型熟悉医院内部的术语、分诊流程和标准话术。

RAG 阶段让模型在回答时检索最新指南、药品说明书和院内知识库。

真正可靠的系统往往不是靠某一种训练技巧，而是把不同机制放在合适的位置。

结语

预训练和微调是一套分工明确的范式：

  • 预训练负责从大规模数据中学习通用能力。
  • 微调负责把通用能力对齐到具体任务。
  • 指令微调让模型更像一个能协作的助手。
  • 参数高效微调让适配成本变得可控。

理解这套范式后，再看大模型训练就不会只看到「堆数据、堆算力」。更本质的问题是：哪些能力应该在底座里形成，哪些行为应该通过微调塑形，哪些知识应该交给外部系统动态提供。
