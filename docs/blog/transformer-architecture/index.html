Transformer 架构详解：从 Attention 到 LLM 的研究脉络

前言

如果说 2012 年的 AlexNet 开启了 CV 的深度学习时代，那么 2017 年的 Transformer 就是 NLP 领域的同一个转折点。今天的大语言模型（GPT-4、Claude、DeepSeek）无一例外都是 Transformer 的后代。

但这篇博客不是讲”怎么调用 API”。我想带你走一遍研究脉络——Transformer 到底解决了什么问题？Self-Attention 为什么能替代 RNN？每一篇关键论文在什么背景下诞生，贡献了什么？

读完你会发现，这八年的研究史有着极其清晰的逻辑链条。

1. 前 Transformer 时代：Seq2Seq 的瓶颈

1.1 用 RNN 做翻译

2014 年，Sutskever 等人提出了 Seq2Seq 架构——用两个 RNN，一个编码器读入源语言，一个解码器逐词生成目标语言。

"我 爱 你" → [Encoder RNN] → 上下文向量 C → [Decoder RNN] → "I love you"

核心问题：无论输入多长，整个句子的信息都被压缩到一个固定长度的上下文向量 C 里。句子一长，C 装不下，翻译质量暴跌。这被称为信息瓶颈。

📄 Sequence to Sequence Learning with Neural Networks
Sutskever, Vinyals, Le. NeurIPS 2014.
提出了 Encoder-Decoder 框架，奠定了神经机器翻译的基础。

1.2 注意力机制的诞生

同年晚些时候，Bahdanau 等人想了一个办法：解码器在生成每个词时，不要只看固定的 C，而是动态地回看输入序列的每个位置，自己决定关注哪些位置。

这就是 Attention 机制的起源：

  • 解码器每生成一个词，计算一个”注意力分数”——当前解码状态和每个编码器隐藏状态的相关性
  • 用 softmax 归一化成权重
  • 加权求和，得到当前时刻专属的上下文向量

这样，生成 “love” 时模型自然会高度关注输入端的 “爱”，而不是平等对待所有词。

📄 Neural Machine Translation by Jointly Learning to Align and Translate
Bahdanau, Cho, Bengio. ICLR 2015.
首次提出 Attention 机制，让解码器动态对齐源语言单词。引用次数超过 3 万。

关键进步：信息瓶颈被打破了。编码器不再需要压缩所有信息到一个向量——解码器可以直接”看到”输入的每个位置。

1.3 痛点仍在

Attention 解决了信息瓶颈，但 RNN 本身的固有问题还在：

  • 串行计算——RNN 必须按时间步一步步算，t 时刻依赖 t-1，无法并行
  • 长程依赖——虽然 Attention 提供了跨位置的直接通路，但编码器本身的 RNN 仍然存在梯度消失问题
  • 训练慢——一个 50 词的句子就要串行跑 50 步

整个领域都在等一个能完全并行的序列建模方案。

2. Transformer 的诞生

2.1 Attention Is All You Need

2017 年 6 月，Google 的 Vaswani 等人扔下一颗炸弹。论文的标题就宣告了一切：Attention Is All You Need——丢掉 RNN，丢掉卷积，只靠 Attention。

📄 Attention Is All You Need
Vaswani, Shazeer, Parmar, Uszkoreit, Jones, Gomez, Kaiser, Polosukhin. NeurIPS 2017.
提出 Transformer 架构，完全基于 Self-Attention，彻底移除循环结构。引用超过 12 万次，是现代 NLP 的基石。

核心洞察：RNN 把序列顺序信息编码到隐藏状态里，代价是串行。但如果我们显式地把位置信息注入输入，就不需要 RNN 的隐藏状态来传递顺序——Self-Attention 可以并行处理整个序列。

架构中有四个需要深入理解的概念，我们把它们一个一个拆开。

3. 四大核心组件

3.1 Self-Attention：序列中的每一个词看每一个词

Self-Attention 是 Transformer 的灵魂。它的核心问题是：

在处理一个词时，句子中的其他词各自有多重要？

以句子 “The cat sat on the mat because it was tired” 为例——“it” 指的是 “cat” 还是 “mat”？人类会根据上下文自然推断，Self-Attention 让模型也能做到这一点。

计算过程：

每个输入 token 生成三个向量：

  • Query (Q)：我要找什么？——当前词在”询问”其他词
  • Key (K)：我能提供什么？——其他词在”应答”
  • Value (V)：我的实际内容是什么？

Q=XWQ,K=XWK,V=XWVQ = XW^Q, \quad K = XW^K, \quad V = XW^VQ=XWQ,K=XWK,V=XWV

其中 WQ,WK,WVW^Q, W^K, W^VWQ,WK,WV 是可学习的参数矩阵，XXX 是输入序列。

注意力分数计算：

Attention(Q,K,V)=softmax(QKTdk)V\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) VAttention(Q,K,V)=softmax(dk​​QKT​)V

逐步拆解：

  1. QKTQK^TQKT：计算每个 Query 和所有 Key 的点积，得到相似度矩阵。结果是一个 n×nn \times nn×n 的矩阵，nnn 是序列长度。
  2. 1dk\frac{1}{\sqrt{d_k}}dk​​1​：除以 Key 维度的平方根做缩放。这是因为点积的方差会随 dkd_kdk​ 增长，softmax 在数值过大时梯度会消失。
  3. softmax：将相似度转为概率分布（权重），确保和为 1。
  4. ×V\times V×V：用权重对 Value 做加权平均。

直觉：输出是输入序列所有词 Value 的加权和，权重由 Query-Key 的相似度决定。“it” 的 Query 和 “cat” 的 Key 相似度更高 → “cat” 的 Value 被赋予更大权重。

3.2 Multi-Head Attention：多个角度同时关注

单一 Attention 可能只捕捉到一种关系（比如主谓关系）。Multi-Head Attention 并行跑多个 Attention，每个头关注不同的模式：

MultiHead(Q,K,V)=Concat(head1,…,headh)WO\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^OMultiHead(Q,K,V)=Concat(head1​,…,headh​)WO

headi=Attention(QWiQ,KWiK,VWiV)\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)headi​=Attention(QWiQ​,KWiK​,VWiV​)

8 个头可能分别学到：主谓关系、修饰关系、同指关系、语义相似度……多头让模型能同时从不同表示子空间捕捉不同类型的依赖。

3.3 Positional Encoding：注入顺序信息

Self-Attention 本身是排列无关的——把句子随机打乱，Attention 的输出（在 softmax 之前）是一样的。但语言显然需要顺序。

Transformer 用位置编码显式地注入位置信息。原始论文使用正弦/余弦函数：

PE(pos, 2i)=sin⁡(pos100002i/dmodel)PE_{(pos,\ 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)PE(pos, 2i)​=sin(100002i/dmodel​pos​)

PE(pos, 2i+1)=cos⁡(pos100002i/dmodel)PE_{(pos,\ 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)PE(pos, 2i+1)​=cos(100002i/dmodel​pos​)

其中 pospospos 是位置，iii 是维度索引。

选择正弦函数的一个好处：对于固定的偏移 kkk，PEpos+kPE_{pos+k}PEpos+k​ 可以表示为 PEposPE_{pos}PEpos​ 的线性变换——这理论上会帮助模型学习相对位置关系。

后来的模型（如 BERT、GPT）更多使用可学习的位置嵌入，效果更好，代价是需要固定最大序列长度。

3.4 Feed-Forward + Residual：非线性与信息保持

每个 Attention 层后面接一个位置独立的前馈网络（通常是两层 MLP，中间用 ReLU/GELU 激活）：

FFN(x)=ReLU(xW1+b1)W2+b2\text{FFN}(x) = \text{ReLU}(xW_1 + b_1)W_2 + b_2FFN(x)=ReLU(xW1​+b1​)W2​+b2​

每个位置独立应用相同的变换。这意味着 Attention 负责”跨位置交流”，FFN 负责”单个位置的深度处理”。

每个子层（Attention 和 FFN）后面都有残差连接（Add）和层归一化（LayerNorm）：

xout=LayerNorm(x+Sublayer(x))x_{out} = \text{LayerNorm}(x + \text{Sublayer}(x))xout​=LayerNorm(x+Sublayer(x))

残差连接让梯度可以走”短路”，是训练深层网络的关键技术——我在上一篇博客里讲过梯度消失的问题，残差连接就是为此而生的。

4. 为什么 Transformer 赢了

维度RNN/LSTMTransformer
并行度串行，t 依赖 t-1完全并行（训练时）
长程依赖梯度消失，理论 > 实践O(1) 的 Attention 通路
训练速度慢（无法利用 GPU 并行）快（矩阵乘法，GPU 友好）
可扩展性差极好（Scaling Law）
可解释性黑盒Attention 权重可视化为对齐图

最后一点尤其重要——Self-Attention 天然产生人类可读的对齐矩阵，这对研究和调试极为有用。

5. 燎原之势：预训练时代

Transformer 论文在 2017 年发表后，接下来的两年里，两个研究分支从根本上重塑了 NLP：

5.1 BERT：双向编码器

📄 BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding
Devlin, Chang, Lee, Toutanova. NAACL 2019.
提出 Masked Language Model 预训练目标，仅使用 Transformer 编码器，在 11 项 NLP 基准上刷新 SOTA。

BERT 的核心思想：

  • 只用 Transformer 编码器——不需要解码器，因为目标不是生成，而是理解
  • Masked Language Model (MLM)：随机遮盖 15% 的输入词，让模型根据上下文双向推断被遮盖的词
  • Next Sentence Prediction (NSP)：判断两句话是否连续（后来被证明不是必需的，RoBERTa 去掉了）

在 BERT 之前，NLP 是”每个任务训练一个专属模型”。BERT 之后，范式变成了：大规模无监督预训练 → 下游任务微调。你在一个巨大的语料库上花几周预训练好一个 BERT，然后在分类、问答、命名实体识别等各种任务上微调几小时就能得到 SOTA。

BERT 的局限：它是双向的，擅长理解但不擅长生成——你没法用 BERT 写文章。

5.2 GPT：单向解码器

OpenAI 走了一条相反的路线。

📄 Improving Language Understanding by Generative Pre-Training (GPT-1)
Radford, Narasimhan, Salimans, Sutskever. 2018.
使用 Transformer 解码器做单向语言模型预训练，开创了生成式预训练范式。

GPT 系列只用 Transformer 解码器，做自回归语言模型——给定前文，预测下一个词。这看起来像是退步（失去了双向信息），但带来的好处是决定性的：

  • 模型天然可以生成文本——只需不断地把自己的输出喂给自己
  • Zero-shot / Few-shot——足够大的 GPT 不需要微调，通过提示词（prompt）就能完成各种任务

Scaling Law 是 GPT 路线最震撼的发现：

📄 Scaling Laws for Neural Language Models
Kaplan, McCandlish, Henighan, et al. 2020.
模型性能随参数量、数据量和计算量的增加呈幂律提升，而非线性饱和。

这意味着：只要给更多的算力和数据，模型会持续变强，没有明显上限。这一发现直接推动了从 GPT-2（1.5B）到 GPT-3（175B）再到 GPT-4 的规模竞赛。

5.3 BERT vs GPT：两条路线

         ┌── 编码器（双向）── BERT ──→ RoBERTa, DeBERTa ──→ 理解任务
        │
Transformer ─┤
        │
        └── 解码器（单向）── GPT-1 ──→ GPT-2 ──→ GPT-3 ──→ GPT-4 ──→ 生成任务
                        └── 语言模型预训练 + Zero-shot

实际上，后来 GPT 路线在规模上取得了全面领先。为什么？GPT-3 证明了：足够大的生成模型可以通过 prompting 完成理解任务（比如翻译、分类），但理解模型永远无法做生成。

5.4 其他重要变体

📄 XLNet: Generalized Autoregressive Pretraining for Language Understanding
Yang, Dai, Yang, Carbonell, Salakhutdinov, Le. NeurIPS 2019.
结合自回归和双向上下文的优点，通过排列语言模型在理解任务上超越 BERT。

📄 T5: Exploring the Limits of Transfer Learning with a Unified Text-to-Text Framework
Raffel, Shazeer, Roberts, Lee, Narang, Matena, Zhou, Li, Liu. JMLR 2020.
将所有 NLP 任务统一为 Text-to-Text 格式，Encoder-Decoder 架构。在 Colossal Clean Crawled Corpus (C4) 上预训练。

📄 ELECTRA: Pre-training Text Encoders as Discriminators Rather Than Generators
Clark, Luong, Le, Manning. ICLR 2020.
用判别器-生成器对抗的方式预训练，在相同算力下显著优于 BERT。

6. LLM 时代：Decoder-Only 一统天下

到了 2023-2024 年，几乎所有主流 LLM 都采用了Decoder-Only Transformer：

模型架构关键创新
GPT-3/4Decoder-OnlyScaling Law, RLHF, 多模态
ClaudeDecoder-OnlyConstitutional AI, RLHF
LLaMADecoder-Only开源, RoPE 位置编码
MistralDecoder-Only滑动窗口注意力, MoE
DeepSeekDecoder-OnlyMoE, 多头潜在注意力

这些模型在架构上没有本质变化——依然是 2017 年 Transformer 解码器的骨架。真正进步来自：

  • 数据规模和质量的提升
  • 训练技术的改进（RLHF, DPO, Constitutional AI）
  • 推理时的工程优化（KV Cache, Flash Attention, Speculative Decoding）
  • 长上下文扩展（RoPE, ALiBi, Ring Attention）

📄 Training Language Models to Follow Instructions with Human Feedback (InstructGPT)
Ouyang, Wu, Jiang, et al. NeurIPS 2022.
提出 RLHF（从人类反馈中强化学习），让模型对齐人类偏好。GPT-3.5/ChatGPT 的核心技术。

📄 LLaMA: Open and Efficient Foundation Language Models
Touvron, Lavril, Izacard, et al. 2023.
证明了用小而精的数据可以在更小参数量下超越 GPT-3，开启了开源 LLM 浪潮。

📄 FlashAttention: Fast and Memory-Efficient Exact Attention
Dao, Fu, Ermon, Rudra, Ré. NeurIPS 2022.
通过 IO-aware 的 tiling 算法将 Attention 的时间/内存从 O(n2)O(n^2)O(n2) 在硬件层面大幅优化，是长上下文的基础设施。

7. Self-Attention 的替代方案

Transformer 的核心瓶颈是 Self-Attention 的二次复杂度——序列长度 nnn 翻倍，计算量翻四倍。这限制了上下文长度。

研究者提出了多种替代方案来打破 O(n2)O(n^2)O(n2)：

方法复杂度核心思想代表论文
Sparse AttentionO(nn)O(n\sqrt{n})O(nn​)每个 token 只关注局部+少量全局Longformer, BigBird
Linear AttentionO(n)O(n)O(n)用核函数改写 softmaxPerformer, Linear Transformer
State Space ModelsO(n)O(n)O(n)用微分方程建模序列Mamba, S4
Sliding WindowO(n⋅w)O(n \cdot w)O(n⋅w)固定窗口局部注意力Mistral

📄 Mamba: Linear-Time Sequence Modeling with Selective State Spaces
Gu, Dao. 2023.
用选择性状态空间模型实现线性复杂度的序列建模，在长序列任务上匹敌甚至超越 Transformer。

目前 Transformer 仍然是主流，但 Mamba 等架构在超长上下文场景（DNA 序列、视频理解）中展现出潜力。未来可能不会有一个”取代 Transformer”的架构，而是 Transformer + 高效 Attention 变体 + SSM 的混合体。
