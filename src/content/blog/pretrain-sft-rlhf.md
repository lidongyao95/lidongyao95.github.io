## 前言

想象你要培养一个人，从零开始到成为某个领域的专业人士。你不会一上来就让他做博士论文，也不会让他读完所有书就直接上岗。更自然的路线是：

```text
中小学：广泛阅读，建立通识基础
大学：选专业，跟导师，做项目
进入社会：接受用户和市场的反馈，持续迭代
```

训练一个大语言模型的过程惊人地相似。从技术视角看，这条路线分为两大阶段：

```text
Pre-Training（无人类参与）  →  Alignment（有人类参与）  →  蒸馏（模型间传递）
                                   ├── SFT：学会听懂指令        teacher → student
                                   └── RLHF：学会判断好坏
```

Pre-Training 让模型从海量文本中建立通识底座；后面两步——SFT 和 RLHF——统称 **Alignment（对齐）**，因为它们都有人类直接参与，目标都是让模型的行为符合人类期望。而 **知识蒸馏** 则是最后一环——把大模型的能力传授给小模型，让昂贵的训练成果变得可部署。

但如果你追问：每个步骤到底在改变模型的什么？为什么顺序不能换？RLHF 里那个「奖励模型」到底是怎么训练出来的？大模型的能力怎么「传」给小模型？很多解释就开始变得含糊。

这篇文章用「培养一个人」的类比，把 Pre-Training、Alignment 和蒸馏完整走一遍。如果你对 Pre-Training 和 SFT 已经有了解，可以直接跳到 [§3 RLHF](#3-rlhf进入社会被真实反馈打磨)——那是本文的重头戏；或者跳到 [§4 知识蒸馏](#4-知识蒸馏老师带学生大模型带小模型) 看老师模型和学生模型的故事。关于预训练和微调的更多细节，也可以参考[《模型训练范式：预训练与微调到底在做什么》](/blog/pretraining-and-finetuning)。

## 1. Pre-Training：中小学阶段的通识教育

### 1.1 一个没有课本的学生

Pre-Training 阶段的模型，就像一个被丢进图书馆的孩子。没有课表，没有考试大纲，唯一的学习方式是：**翻开一本书，盖住下一个词，猜猜它是什么。**

这就是自回归语言模型的训练目标——next-token prediction：

$$\mathcal{L}_{\text{PT}} = -\sum_{t=1}^{T} \log P(x_t \mid x_{<t}; \theta)$$

模型看到前面的 token 序列 $x_{<t}$，预测下一个 token $x_t$。预测错了就更新参数，如此反复数万亿次。

听起来很简单，但仔细想想：一个人如果要把世界上所有文本的「下一个词」都猜对，他不得不学会什么？

- 语法和句法结构（不然连句子都拼不对）
- 事实和常识（「中国的首都是___」，不知道就猜不对）
- 推理模式（数学题的下一步、代码的下一行）
- 文体切换（论文和段子的接续方式完全不同）

这就是 Pre-Training 的本质：**模型不是在「背知识」，而是在学习一种能压缩所有文本分布的表示能力。** 关于这个阶段的更多细节，包括表示学习和参数效率微调，可以参考[预训练与微调](/blog/pretraining-and-finetuning)的前两节。

### 1.2 通识教育的天花板

一个读遍图书馆的人，知识量一定很大，但你让他去当医生，他大概率不行。问题不在于知识不够，而在于：

- **没有目标感**：他擅长续写文本，但你问他一个问题，他可能续写一段类似问题的网页内容，而不是回答问题。
- **没有格式意识**：你让他「用三点总结」，他可能根本不理你。
- **不知道什么该说什么不该说**：他从互联网上学到的，包括好的和坏的。

Pre-Training 给模型建立了一个很高的能力上限，但这个上限怎么被调用、往哪个方向释放，还需要后续的 **Alignment** 阶段来引导。

用一个公式来概括两大阶段的关系：

$$\text{最终能力} = \underbrace{\text{Pre-Training}}_{\text{能力上限（无人监督）}} \times \underbrace{\overbrace{\text{SFT}}^{\text{能力调用}} \times \overbrace{\text{RLHF}}^{\text{行为对齐}}}_{\text{Alignment（人类参与）}}$$

Pre-Training 是底座，Alignment 决定底座怎么被使用、往什么方向释放。SFT 让模型学会听懂任务，RLHF 让模型的行为符合人类期望。底座再强，对齐做不好，模型照样不可用。

## 2. SFT：Alignment 第一步，大学阶段的专业训练

### 2.1 选专业，跟导师

如果 Pre-Training 是通识教育，SFT（Supervised Fine-Tuning）就是大学里的专业训练。

一个大学生要做什么？选一个方向，跟一个导师，做一批有标准答案的项目。SFT 完全一样——我们给模型准备一批高质量的「输入-输出」对：

```text
用户：用通俗语言解释什么是梯度下降。
助手：梯度下降可以理解为在山坡上找最低点。你站在山上，
每一步都往最陡的下坡方向走，最终就能走到谷底……
```

模型的损失函数变成了：只在输出 token 上计算交叉熵。

$$\mathcal{L}_{\text{SFT}} = -\sum_{t \in \text{output}} \log P(y_t \mid x, y_{<t}; \theta)$$

注意这里的区别：Pre-Training 在所有 token 上算 loss（因为整段文本都是学习对象），SFT 只在「助手回复」的 token 上算 loss。用户的问题只是条件，不是学习目标。

### 2.2 SFT 让模型学会了什么

经过 SFT，模型的行为发生了一次质变：

**从「续写者」变成「对话者」。** 它开始理解：当用户说「请解释」时，你应该真的解释；当用户说「总结」时，你应该给出摘要；当用户问了一个问题，你应该回答，而不是续写一段类似问题的文本。

但 SFT 有一个根本性的局限：**它只能教模型模仿标准答案，不能教模型判断「什么是好的回答」。**

想想一个大学生。如果他只做过有标准答案的习题，从没接受过导师的反馈、同学的讨论、审稿人的批评，他能成为一个好的研究者吗？大概不行。他知道怎么「做题」，但不知道怎么「判断好坏」。

这正是 RLHF 要解决的问题。

### 2.3 一个意外发现：SFT 甚至不需要「指令」

前面说 SFT 需要高质量的「指令-回复」对，但 2024 年的一项研究对这个假设提出了挑战。

Xie、Syu 和 Lee 在 **Non-instructional Fine-tuning（NIF）** 的工作中做了一个反直觉的实验：他们用随机网页文本（OpenWebText）的前半段作为「输入」，让 GPT-4 续写后半段作为「输出」，构造了一批**完全不含任何指令**的训练数据。没有「请解释」，没有「请总结」，没有任何人类编写的 prompt。

```text
传统 SFT 数据:
  输入: "请用通俗语言解释什么是梯度下降。"     ← 明确的指令
  输出: "梯度下降可以理解为在山坡上找最低点……"  ← 对指令的回复

NIF 数据:
  输入: "The history of the Roman Empire can be divided into..."  ← 随机网页前半段
  输出: "...three major periods: the Republic, the Principate,..."  ← GPT-4 续写后半段
```

结果令人惊讶：在这些**非指令数据**上微调后的 base model（Mistral、LLaMA 等）竟然获得了指令遵循能力。它们从未见过「请做 X」这种格式，但训练后你给它一个指令，它就能执行。

这个发现为什么重要？它暗示了 **指令遵循能力可能不是被「教」出来的，而是被「唤醒」的。** 模型在 Pre-Training 阶段已经见过足够多的指令-回复模式（互联网上到处都是问答论坛、教程、FAQ），SFT 的真正作用或许不是「教会」模型听懂指令，而是**调整模型的分布，让指令遵循这种已经存在于参数中的能力被激发出来**。

NIF 甚至可以叠加在 RLHF 之上进一步提升效果——他们最大的模型在 Arena Hard 上达到了接近 GPT-4-turbo 的水平。这意味着从 Pre-Training 到 Alignment 的整条流水线中，SFT 这一步的门槛可能比我们想象的要低得多：你甚至不需要精心设计指令数据，只要有足够好的「续写」数据就够了。

## 3. RLHF：Alignment 第二步，进入社会被真实反馈打磨

### 3.1 为什么 SFT 不够

让我们先认真回答一个问题：SFT 之后的模型已经能对话了，为什么还需要 RLHF？

核心原因在于 **SFT 只能传递行为，不能传递偏好。**

举个例子。你让模型写一篇产品介绍，SFT 数据里有 10 个范例。模型学会了写产品介绍——但它同时从互联网上学到了夸大宣传、虚假承诺、啰嗦重复的写法。在 SFT 的框架下，只要训练数据够多、质量够高，模型的表现就会不错。但问题是：

- 高质量数据是有限的。你不可能为每种场景都准备完美的示范。
- 有些东西很难用「示范」来教。什么叫「语气恰当」？什么叫「有帮助但不啰嗦」？什么叫「拒绝时保持礼貌」？
- 模型可能在训练数据上表现很好，但遇到分布外的输入就原形毕露。

**SFT 像考试——有标准答案就能学；RLHF 像工作——没有标准答案，只有「这个比那个好」。**

### 3.2 偏好数据：不是「正确答案」，而是「这个比那个好」

RLHF 的第一步是收集偏好数据。做法很直觉：

让模型对同一个 prompt 生成多个不同的回复，然后让人类标注者选择更好的那个。

```text
Prompt: "如何评价这家餐厅？"

回复 A: "这家餐厅味道还行，价格也还可以，服务一般般。"
回复 B: "这家餐厅的菜品口味偏清淡，招牌菜是松鼠鱼，
         人均消费 80 元左右，服务响应速度较快。"

标注者选择: B > A
```

注意，标注者不需要写出「正确答案」，只需要做 **比较**。这比写标准答案容易得多。你可以很轻松地说出「这个回答比那个好」，但让你从零写出一个完美回答，难度要大得多。

收集大量这样的比较数据后，我们就有了一个偏好数据集：$(x, y_w, y_l)$，其中 $y_w$ 是偏好的回复（winner），$y_l$ 是不偏好的回复（loser）。

### 3.3 奖励模型：把人类偏好变成一个可微的分数

有了偏好数据，下一步是训练一个**奖励模型（Reward Model）**。

奖励模型的目标很明确：给定一个 prompt 和一个回复，打一个分。这个分数要能反映人类的偏好排序。

$$r_\phi(x, y) \in \mathbb{R}$$

训练奖励模型用的是 Bradley-Terry 偏好模型。核心思想是：如果人类认为 $y_w$ 比 $y_l$ 好，那奖励模型给 $y_w$ 的分数应该比 $y_l$ 高。损失函数是：

$$\mathcal{L}_{\text{RM}} = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma \left( r_\phi(x, y_w) - r_\phi(x, y_l) \right) \right]$$

直觉很好理解：$r_\phi(x, y_w) - r_\phi(x, y_l)$ 越大，说明奖励模型越能区分好坏，$\sigma(\cdot)$ 越接近 1，$\log$ 越接近 0，loss 越小。如果奖励模型分不清好坏，差值接近 0，$\sigma(0) = 0.5$，loss 就很大。

奖励模型通常从 SFT 模型初始化，把最后一层改成标量输出。它的参数量一般和 SFT 模型相当或略小，因为它的任务不是生成文本，而是评估文本质量。

```text
┌──────────────────────────────────────────────────────┐
│              RLHF 训练流水线                           │
│                                                      │
│  ┌─────────┐    偏好数据    ┌──────────┐             │
│  │ SFT 模型 │ ──────────→  │ 奖励模型  │             │
│  │ π_sft   │   训练 RM     │ r_φ(x,y) │             │
│  └────┬────┘              └─────┬────┘             │
│       │                        │                    │
│       │ 初始化策略模型           │ 提供奖励信号        │
│       ↓                        ↓                    │
│  ┌─────────┐   PPO 优化    ┌──────────┐             │
│  │ 策略模型 │ ←─────────── │ 奖励模型  │             │
│  │ π_θ     │   打分+KL约束 │ r_φ(x,y) │             │
│  └─────────┘              └──────────┘             │
└──────────────────────────────────────────────────────┘
```

### 3.4 PPO：用奖励信号优化语言模型

有了奖励模型，下一步就是用它来优化语言模型。这一步用的是 **PPO（Proximal Policy Optimization）**，一种强化学习算法。

把语言模型想象成一个策略（policy）：输入是 prompt $x$，动作是生成的回复 $y$，奖励是 $r_\phi(x, y)$。强化学习的目标是最大化期望奖励。

但这里有一个关键问题：**如果直接最大化奖励模型的分数，模型会找到奖励模型的漏洞。** 就像一个员工如果只按 KPI 考核，他可能学会刷数据而不是真正做好工作。

解决方案是加一个 KL 散度约束，让优化后的模型不要偏离 SFT 模型太远：

$$\max_{\theta} \; \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(\cdot|x)} \left[ r_\phi(x, y) - \beta \log \frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)} \right]$$

其中 $\pi_{\text{ref}}$ 是 SFT 模型（参考策略），$\beta$ 控制 KL 惩罚的强度。

这个公式的直觉是：

- **第一项** $r_\phi(x, y)$：鼓励生成奖励模型打分高的回复（做「好」的回答）
- **第二项** $\beta \log \frac{\pi_\theta}{\pi_{\text{ref}}}$：惩罚偏离 SFT 模型太远的行为（别走极端）

PPO 的训练过程大致是这样的：

1. 用当前策略 $\pi_\theta$ 对一批 prompt 生成回复
2. 用奖励模型 $r_\phi$ 给每个回复打分
3. 用优势函数（advantage function）估计每个 token 的贡献
4. 用截断（clipping）机制更新策略，防止单步更新过大
5. 重复以上步骤

PPO 的核心截断机制：

$$L^{\text{CLIP}}(\theta) = \mathbb{E}_t \left[ \min \left( \rho_t \hat{A}_t, \; \text{clip}(\rho_t, 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right]$$

其中 $\rho_t = \frac{\pi_\theta(a_t|s_t)}{\pi_{\text{old}}(a_t|s_t)}$ 是新旧策略的概率比。截断保证每次更新的幅度不超过 $\epsilon$（通常 0.2），避免策略在一步之内跑太远。

### 3.5 RLHF 在工程上有多难

RLHF 在论文里看起来很优雅，工程落地时却是一个不折不扣的系统工程挑战。

**四个模型同时在显存里。** PPO 训练需要同时维护：策略模型（actor）、参考模型（reward baseline）、奖励模型（reward model）、价值模型（critic / value model）。即使对 7B 的模型，也需要 4 × 7B = 28B 参数的显存。这也是为什么 DeepSpeed ZeRO、FSDP 等分布式训练框架在 RLHF 阶段至关重要。

**奖励模型的漂移。** 随着策略模型的优化，它生成的回复越来越擅长「骗」奖励模型打高分，但实际质量未必提升。这叫 **reward hacking**，需要定期用新的人类偏好数据更新奖励模型。

**训练不稳定。** PPO 对超参数非常敏感。学习率、KL 系数 $\beta$、clip 范围、batch size 的微小变化都可能导致训练崩溃。

### 3.6 DPO：跳过奖励模型的捷径

RLHF 的工程复杂度催生了一个自然的追问：**能不能绕过奖励模型，直接从偏好数据优化策略？**

**DPO（Direct Preference Optimization）** 给出了一个漂亮的回答。Rafailov 等人在 2023 年的论文中证明，RLHF 的目标函数有一个封闭形式的等价变换：

$$\mathcal{L}_{\text{DPO}}(\theta) = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)} \right) \right]$$

这个公式看起来有点吓人，但直觉非常清晰：

- 对偏好回复 $y_w$：让 $\pi_\theta$ 相对于 $\pi_{\text{ref}}$ 生成它的概率变大
- 对非偏好回复 $y_l$：让 $\pi_\theta$ 相对于 $\pi_{\text{ref}}$ 生成它的概率变小
- $\beta$ 控制调整幅度，防止偏离太远

DPO 的精妙之处在于：它把奖励模型和策略优化两步合为一步。你只需要 SFT 模型和偏好数据，就可以直接训练最终策略。不需要单独训练奖励模型，不需要跑 PPO，不需要四个模型同时在显存里。

| 方法 | 需要奖励模型 | 训练复杂度 | 核心风险 |
|------|:---:|:---:|------|
| PPO-based RLHF | 是 | 高（4 模型并行） | Reward hacking、训练不稳定 |
| DPO | 否 | 低（1 模型 + 参考模型） | 偏好数据质量直接决定效果 |
| RLHF + 过程奖励 | 是 | 很高 | 过程标注成本 |

DPO 的代价是什么？因为它没有显式的奖励模型，所以无法在训练过程中动态采样和评估新回复。它更像是「离线学习」——从已有的偏好数据中学，而 PPO 是「在线学习」——不断生成新回复并评估。在某些场景下，PPO 的上限更高；但在大多数工程实践中，DPO 的简洁性让它在效果-成本权衡中胜出。

## 4. 知识蒸馏：老师带学生，大模型带小模型

### 4.1 一个现实问题：大模型太贵了

经过 Pre-Training → SFT → RLHF 完整流水线训练出来的模型，能力确实很强。但一个 70B 甚至更大参数的模型，推理一次需要几十张 GPU，延迟高、成本大，根本没法直接部署到生产环境。

回到培养的类比：你花了很多年、很多资源培养出了一个顶级专家，但社会需要的不只是一个专家——你需要一大批合格的专业人士。怎么办？让专家去带学生。

这就是 **知识蒸馏（Knowledge Distillation）** 的核心动机：**用一个强大的「老师模型」（teacher）来训练一个轻量的「学生模型」（student），让小模型以低成本继承大模型的核心能力。**

### 4.2 经典蒸馏：学到的不只是答案，还有「犹豫」

Hinton 等人在 2015 年提出的知识蒸馏，思路非常优雅。

传统的训练方式是用 hard label——正确答案是「猫」，标签就是 $[0, 0, 1]$。但一个训练好的老师模型输出的不是这种非黑即白的标签，而是一个概率分布，比如：

```text
Hard label:  [0,   0,   1,   0  ]   → 猫
Soft label:  [0.02, 0.05, 0.85, 0.08] → 85% 猫, 8% 狗, 5% 狐狸, 2% 兔子
```

老师模型把「猫」和「狐狸」的相似度、「猫」和「狗」的混淆模式这些**暗知识（dark knowledge）** 编码在了 soft label 里。学生模型如果只学 hard label，就丢掉了这些信息；如果学 soft label，就能继承老师对类别间关系的理解。

为了让暗知识更明显，蒸馏引入 **温度参数 $T$** 来「软化」softmax 输出：

$$q_i = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)}$$

$T=1$ 是标准 softmax，$T$ 越大分布越平滑，类别间的相对关系越清晰。学生模型的损失函数是 soft label 和 hard label 的加权组合：

$$\mathcal{L}_{\text{KD}} = \alpha \cdot \mathcal{L}_{\text{CE}}(y, \hat{y}) + (1-\alpha) \cdot T^2 \cdot \text{KL}(p^T_{\text{teacher}} \| p^T_{\text{student}})$$

第一项是标准的交叉熵（学正确答案），第二项是 KL 散度（学老师的分布形状）。$T^2$ 的缩放是为了补偿温度对梯度的衰减。

经典蒸馏在小模型（如图像分类的 CNN）上效果很好。但到了大语言模型时代，情况变了。

### 4.3 LLM 时代的蒸馏：从 soft label 到数据蒸馏

LLM 的蒸馏和经典蒸馏有一个根本区别：**你没法直接拿一个 70B 模型的 logits 去训练一个 7B 模型。** 两个模型的 vocabulary 可能不同，参数空间不兼容，logits 的含义也不一样。

所以 LLM 时代的蒸馏转向了一个更直觉的方向：**让老师模型生成高质量的训练数据，再让学生模型用这些数据做 SFT。**

```text
经典蒸馏:  teacher logits → student logits（直接对齐输出分布）
LLM 蒸馏:  teacher outputs → student SFT data（通过数据间接传递知识）
```

这条路线有几个里程碑：

**Self-Instruct（2022）**：Wang 等人让 GPT-3 自己生成指令-回复对，再用这些数据微调一个小模型。核心循环是：给 GPT-3 少量种子指令，让它生成新指令，再让它回答这些指令，把结果作为训练数据。这相当于让老师不仅回答问题，还自己出考题。

**Alpaca 和 Vicuna（2023）**：Stanford 和 UC Berkeley 的团队分别用 Self-Instruct 的思路，从 GPT-3.5/4 蒸馏出 7B 和 13B 的模型。Alpaca 用 52K 条 GPT-3.5 生成的指令数据微调 LLaMA，训练成本不到 600 美元。Vicuna 则用 ShareGPT 上用户和 ChatGPT 的真实对话数据，质量更高。

**DeepSeek-R1 蒸馏（2024）**：这是 LLM 蒸馏的一个新范式。DeepSeek 先通过 RL 训练出一个具有强推理能力的 R1-Zero（老师），再用 R1-Zero 生成大量高质量推理数据，蒸馏到 7B、14B、32B 等不同规模的学生模型。蒸馏后的 DeepSeek-R1-Distill-Qwen-32B 在数学和代码推理任务上甚至超过了很多完整 RLHF 训练的大模型。

### 4.4 蒸馏的本质：能力传递还是行为模仿？

这里有一个值得思考的问题：**蒸馏到底传递了什么？**

在经典蒸馏中，学生学的是老师的 soft label 分布——这包含了老师对输入空间的结构化理解（哪些类别容易混淆、哪些特征重要）。这是一种**能力传递**。

在 LLM 蒸馏中，学生学的是老师的文本输出——这更多是一种**行为模仿**。学生模仿老师的回答风格、推理路径和知识组织方式，但不一定继承了老师底层的「理解」。

一个有意思的证据是：蒸馏后的学生模型在老师训练分布内的任务上表现很好，但在需要深层推理或创造性泛化的任务上往往会掉队。这就像一个学生如果只抄过老师的答案而没有真正理解解题思路，遇到变形题就会卡壳。

不过 DeepSeek-R1 的蒸馏实践给出了一个更乐观的信号：如果老师模型的能力本身是通过 RL 获得的（而非纯 SFT），蒸馏的效果会显著更好。这可能是因为 RL 训练出的推理模式更容易通过输出行为被「外化」，从而更容易被学生模仿。

### 4.5 蒸馏与 RLHF 中的师生隐喻

其实，回顾整个 Pre-Training → Alignment 的流水线，「师生」关系无处不在，只是形式不同：

| 阶段 | 老师是谁 | 学生是谁 | 老师教了什么 |
|------|----------|----------|--------------|
| SFT | 人类标注者 | Pre-Trained 模型 | 任务执行的标准示范 |
| RLHF (PPO) | 奖励模型 + 参考模型 | 策略模型 | 好坏判断 + 行为边界 |
| DPO | 偏好数据（隐含人类偏好） | SFT 模型 | 直接的行为偏好 |
| 知识蒸馏 | 大模型 (teacher) | 小模型 (student) | 能力与行为的综合传递 |

SFT 的老师是人类，教的是「怎么做」；RLHF 的老师也是人类（通过奖励模型代理），教的是「什么好」；蒸馏的老师是另一个模型，教的是「像我这样做」。

三种师生关系对应了三种不同的学习范式：**模仿示范、接受反馈、师承传授。** 这和人类学习的经验也是吻合的——你在学校跟老师学知识（SFT），在社会上被用户和同行反馈（RLHF），在团队里被前辈带（蒸馏）。

## 5. 全景：从 Pre-Training 到 Alignment 到蒸馏

现在我们把两大阶段、三个步骤放在一起看：

| 阶段 | 所属 | 类比 | 数据来源 | 训练信号 | 改变了模型的什么 |
|------|------|------|----------|----------|------------------|
| Pre-Training | 预训练 | 中小学通识教育 | 互联网文本（万亿 token） | next-token prediction | 知识和语言能力 |
| SFT | Alignment | 大学专业训练 | 人工标注的指令-回复对 | 交叉熵（仅输出 token） | 任务执行能力 |
| RLHF / DPO | Alignment | 社会反馈打磨 | 人类偏好比较数据 | 奖励信号 / 偏好 loss | 行为对齐和价值偏好 |
| 知识蒸馏 | 部署 | 师承传授 | 老师模型的输出 | SFT loss（模仿老师行为） | 能力压缩与迁移 |

四个步骤的数据规模递减、标注成本递减：

```text
Pre-Training:  ~10T tokens, 几乎无标注成本（自动构造）
SFT:           ~100K 条,    人工撰写高质量回复
RLHF:          ~50K 对,     人工比较和排序
知识蒸馏:       ~100K 条,    老师模型自动生成（几乎零标注成本）
```

前三个步骤的杠杆效应递增：Pre-Training 决定了能力天花板，SFT 决定了能力的可用性，RLHF 决定了用户愿不愿意用。蒸馏则是最后一步——把前三步的成果压缩到一个可部署的小模型里。一个底座很强但没做 Alignment 的模型，可能输出有害内容或答非所问；一个做了 Alignment 但底座较弱的模型，至少安全且有用，只是深度有限；而蒸馏让这种「安全且有用」变得廉价可得。

### 5.1 一个常见误解：RLHF 不是「教模型新知识」

很多人以为 RLHF 能让模型学到 SFT 阶段没学到的知识。这是一个误解。

RLHF 改变的是模型的**行为偏好**，而不是知识储备。它不会让模型突然知道 SFT 阶段不知道的事实，但会让模型学会：什么时候该详细回答，什么时候该简洁；什么时候该表达不确定性；什么时候该拒绝。

回到类比：一个大学生进入社会后，用户反馈不会突然教会他大学没学过的微积分。但反馈会教会他：汇报时要先说结论再说过程、客户投诉时要先共情再解决、写邮件时不要太啰嗦。

```text
知识层面:  Pre-Training > SFT >> RLHF
行为层面:  RLHF > SFT > Pre-Training
```

### 5.2 从 RLHF 到 Constitutional AI 到 GRPO

RLHF 不是对齐的终点，而是一个起点。近年来出现了一系列改进：

**Constitutional AI（CAI）**：Anthropic 提出用一组原则（constitution）让模型自我批评和修正，减少对人类标注的依赖。思路是：先让模型生成回复，再让它自己根据原则修改，最后用修改后的数据做 RLHF。

**GRPO（Group Relative Policy Optimization）**：DeepSeek 在 R1 中使用的方法。核心思想是对同一个 prompt 生成一组回复，用组内相对排名替代单独的奖励模型，进一步简化了 RLHF 流程。

**在线 DPO / iterative DPO**：在 DPO 的基础上，用当前模型不断采样新回复、收集新偏好、继续训练，弥补 DPO 离线学习的局限。

这些方法的共同方向是：**降低标注成本、提高对齐质量、减少训练不稳定性。** 对齐领域仍在快速演进，但 Pre-Training + Alignment（SFT → RLHF）的框架，仍然是理解所有后续工作的基础。

## 结语

回到开头的类比。培养一个人和训练一个大模型，底层逻辑是相似的：先广泛地学，建立足够宽的知识底座；再有针对性地学，掌握具体的任务技能；然后在真实世界的反馈中迭代，让自己的行为符合社会期望。最后，把自己积累的能力传授给下一代——这是知识蒸馏在做的事，也是人类文明延续的基本方式。

但有一点值得深思：人类在每个步骤中是有主体性的——你可以选择读什么书、跟什么导师、接受什么反馈、带什么学生。而模型的每一步都是由人类设计的：Pre-Training 的数据是人选的，SFT 的标注是人做的，RLHF 的偏好是人定义的，蒸馏的老师也是人挑的。

**所谓「对齐」，本质上是把人类的偏好编码进模型的参数里。** 这意味着对齐的质量上限，取决于我们对「什么是好的行为」这件事的理解深度。如果人类自己都说不清什么是对的，模型也很难被对齐到「对」的方向。

而蒸馏则引出了另一个问题：当学生模型越来越多地从老师模型学习，而不是从人类数据学习时，对齐的质量会不会逐代衰减？这正是「模型坍缩（model collapse）」研究在追问的事情。当师承链条越来越长，最初来自人类的那些偏好信号，会不会在传递中逐渐失真？

这可能是大模型训练中最有意思的哲学问题：我们训练模型的过程，也是一次对自身价值观的审视。而当我们让模型去训练模型时，我们还需要确保这种审视不会在传递中走样。
